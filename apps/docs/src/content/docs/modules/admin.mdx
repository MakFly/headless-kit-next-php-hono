## Overview

The Admin module provides a comprehensive back-office: user and role management (RBAC), product and order CRUD, revenue analytics, customer management, review moderation, and customer segments.

<Aside>
  All admin endpoints require the `admin` role or specific permissions. Access is controlled via middleware (Laravel), voters (Symfony), or middleware (Hono).
</Aside>

## API Endpoints

### RBAC

| Method | Endpoint | Permission | Description |
| ------ | -------- | ---------- | ----------- |
| `GET` | `/api/v1/admin/users` | `users:read` | List all users |
| `GET` | `/api/v1/admin/users/{id}` | `users:read` | Show user details |
| `POST` | `/api/v1/admin/users/{id}/roles` | `roles:manage` | Assign role to user |
| `DELETE` | `/api/v1/admin/users/{id}/roles/{role}` | `roles:manage` | Remove role from user |
| `GET` | `/api/v1/admin/roles` | `roles:read` | List roles |
| `POST` | `/api/v1/admin/roles` | `roles:manage` | Create role |
| `GET` | `/api/v1/admin/permissions` | `permissions:read` | List permissions |
| `POST` | `/api/v1/admin/roles/{id}/permissions` | `permissions:manage` | Sync permissions |

## Backend Implementation

The Admin feature has the most actions of any module (~30 per backend):

- **Laravel**: `app/Features/Admin/Actions/` — invokable controllers
- **Symfony**: `src/Feature/Admin/Controller/Rbac/` and `src/Feature/Admin/Controller/Shop/` — invokable controllers
- **Hono**: `src/features/admin/` — 5-file convention
