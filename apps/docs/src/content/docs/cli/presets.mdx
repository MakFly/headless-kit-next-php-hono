## What are Presets?

Presets are pre-configured application templates that bundle the right pages, components, and backend modules for a specific use case. When you select a preset during `bunx create-headless-app`, it overlays additional files on top of the base template.

## Available Presets

### E-commerce

Full online store with shopping cart, checkout, and admin dashboard.

| Layer | What's Included |
| ----- | --------------- |
| Frontend | Product catalog, cart sidebar, checkout flow, order history, admin dashboard |
| Backend modules | Auth, Shop, Cart, Orders, Admin |
| Components | Product cards, cart drawer, order summary, data tables |

**Best for:** Online stores, marketplaces, product-based businesses.

### SaaS

Multi-tenant SaaS platform with billing, team management, and admin.

| Layer | What's Included |
| ----- | --------------- |
| Frontend | Landing page, org dashboard, billing page, team management, settings |
| Backend modules | Auth, SaaS (Orgs, Plans, Subscriptions, Invoices, Teams), Admin |
| Components | Org switcher, plan cards, invoice table, team list |

**Best for:** B2B SaaS products, multi-tenant platforms, subscription businesses.

### Admin

Back-office dashboard with RBAC, analytics, and data management.

| Layer | What's Included |
| ----- | --------------- |
| Frontend | Dashboard with stats, user management, role management, permissions |
| Backend modules | Auth, Admin (RBAC, Dashboard, Analytics) |
| Components | Data tables, stat cards, role editor, permission matrix |

**Best for:** Internal tools, admin panels, CMS backends.

### Landing

Minimal marketing site with authentication.

| Layer | What's Included |
| ----- | --------------- |
| Frontend | Landing page, login, register |
| Backend modules | Auth |
| Components | Auth forms, minimal layout |

**Best for:** MVPs, landing pages with auth, simple apps that will grow.

<Aside type="note">
  Support Chat and AI Assistant are available as **additional modules** during the CLI prompt, regardless of which preset you choose. They are not bundled in any preset by default.
</Aside>

## Module Dependencies

Some additional modules have dependencies on other modules:

| Module | Requires |
| ------ | -------- |
| Multi-tenant | — |
| Billing | Multi-tenant |
| Team management | Multi-tenant |
| Org-level RBAC | Multi-tenant + Team |
| Support chat | — |
| AI Assistant | — |

The CLI automatically selects required dependencies when you pick a module that depends on them.

## Customizing After Scaffold

Presets are just starting points. After scaffolding:

- Add or remove pages from `apps/web/src/app/` (Next.js) or `apps/web/src/routes/` (TanStack Start)
- Add backend features by creating new feature directories (see [VSA guide](/architecture/vertical-slice/))
- Modify the admin dashboard to fit your data model
- Adjust component styling via Tailwind and shadcn/ui theme

<Aside type="tip">
  All presets use the same Vertical Slice Architecture across all backends. See the [VSA guide](/architecture/vertical-slice/) for how to add custom features.
</Aside>
