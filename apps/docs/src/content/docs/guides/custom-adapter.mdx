## Overview

If you want to connect Headless Kit to a backend not included out of the box (e.g., Django, Spring Boot, Fastify), you need to implement a custom adapter. Adapters translate between the standardized `AuthAdapter` interface and your backend's specific API.

## The AuthAdapter Interface

Every adapter must implement:

```typescript
type AuthAdapter = {
  login(credentials: LoginRequest): Promise<AuthResponse>;
  register(data: RegisterRequest): Promise<AuthResponse>;
  logout(): Promise<void>;
  refresh(request?: RefreshTokenRequest): Promise<AuthResponse>;
  getUser(): Promise<NormalizedUser | null>;
  getOAuthProviders(): Promise<string[]>;
  getOAuthUrl(provider: string): Promise<string>;
  sendMagicLink?(email: string): Promise<void>;
  storeTokens(tokens: TokenStorage): Promise<void>;
  clearTokens(): Promise<void>;
  getAccessToken(): Promise<string | null>;
};
```

## Building a Custom Adapter

<Steps>

1. **Extend BaseAdapter**

   `BaseAdapter` handles token storage, cookie management, and auth header injection. You only need to implement the abstract methods:

   ```typescript
   // apps/web/src/lib/adapters/custom/adapter.ts
   import { BaseAdapter, AdapterError } from '../base-adapter';
   import type { BaseAdapterDeps } from '../base-adapter';
   import type {
     AdapterConfig,
     LoginRequest,
     RegisterRequest,
     RefreshTokenRequest,
     AuthResponse,
     NormalizedUser,
   } from '../types';
   import {
     transformLoginResponse,
     transformUserResponse,
   } from './transformer';

   export class CustomAdapter extends BaseAdapter {
     constructor(config: AdapterConfig, deps: BaseAdapterDeps) {
       super(config, deps);
     }

     async login(credentials: LoginRequest): Promise<AuthResponse> {
       const response = await this.makeRequest<unknown>(
         'POST',
         '/auth/login',
         { body: credentials, includeAuth: false }
       );

       const authResponse = transformLoginResponse(response);
       await this.storeTokens(authResponse.tokens);
       return authResponse;
     }

     async register(data: RegisterRequest): Promise<AuthResponse> {
       const response = await this.makeRequest<unknown>(
         'POST',
         '/auth/register',
         { body: data, includeAuth: false }
       );

       const authResponse = transformLoginResponse(response);
       await this.storeTokens(authResponse.tokens);
       return authResponse;
     }

     async logout(): Promise<void> {
       try {
         await this.makeRequest('POST', '/auth/logout');
       } finally {
         await this.clearTokens();
       }
     }

     async refresh(request?: RefreshTokenRequest): Promise<AuthResponse> {
       const refreshToken =
         request?.refresh_token || (await this.getRefreshToken());

       const response = await this.makeRequest<unknown>(
         'POST',
         '/auth/refresh',
         { body: refreshToken ? { refresh_token: refreshToken } : undefined }
       );

       const authResponse = transformLoginResponse(response);
       await this.storeTokens(authResponse.tokens);
       return authResponse;
     }

     async getUser(): Promise<NormalizedUser | null> {
       try {
         const response = await this.makeRequest<unknown>('GET', '/auth/me');
         return transformUserResponse(response);
       } catch (error) {
         if (error instanceof AdapterError && error.statusCode === 401) {
           return null;
         }
         throw error;
       }
     }

     async getOAuthProviders(): Promise<string[]> {
       return []; // Implement if your backend supports OAuth
     }

     async getOAuthUrl(provider: string): Promise<string> {
       throw new Error(`OAuth not supported for provider: ${provider}`);
     }
   }
   ```

2. **Create a Transformer**

   The transformer normalizes your backend's response format into Headless Kit's standard types:

   ```typescript
   // apps/web/src/lib/adapters/custom/transformer.ts
   import type { AuthResponse, NormalizedUser } from '../types';

   export function transformLoginResponse(raw: unknown): AuthResponse {
     const data = raw as Record<string, unknown>;

     return {
       user: transformUserResponse(data.user || data),
       tokens: {
         access_token: (data.access_token || data.token) as string,
         refresh_token: data.refresh_token as string | undefined,
         expires_in: (data.expires_in || 3600) as number,
         token_type: 'Bearer',
       },
     };
   }

   export function transformUserResponse(raw: unknown): NormalizedUser {
     const data = raw as Record<string, unknown>;

     return {
       id: data.id as string | number,
       email: data.email as string,
       name: (data.name || data.username || '') as string,
       email_verified_at: (data.email_verified_at || null) as string | null,
       roles: [],       // Map from your backend's role format
       permissions: [],  // Map from your backend's permission format
     };
   }
   ```

3. **Export from the adapters module**

   ```typescript
   // apps/web/src/lib/adapters/custom/index.ts
   export { CustomAdapter } from './adapter';
   ```

   Add to the main adapters barrel:

   ```typescript
   // apps/web/src/lib/adapters/index.ts
   export * from './custom';
   ```

4. **Register in the BFF proxy**

   Add your backend type to the proxy configuration:

   ```typescript
   // Update BffNextjsConfig backends to include your custom backend
   backends: {
     laravel: { baseUrl: '...' },
     symfony: { baseUrl: '...' },
     node: { baseUrl: '...' },
     custom: { baseUrl: 'http://localhost:9000' },
   },
   ```

</Steps>

## CookieStore Injection

The `BaseAdapter` receives a `CookieStore` through dependency injection. This allows the same adapter to work in different environments:

```typescript
// CookieStore type from apps/web/src/lib/adapters/types.ts

// Cookie store implementation is in the proxy route handler

// In Express
const cookieStore: CookieStore = {
  get: (name) => {
    const value = req.cookies[name];
    return value ? { name, value } : undefined;
  },
  set: (options) => {
    res.cookie(options.name, options.value, {
      httpOnly: options.httpOnly,
      secure: options.secure,
      sameSite: options.sameSite,
      path: options.path,
      maxAge: options.maxAge ? options.maxAge * 1000 : undefined,
    });
  },
  delete: (name) => {
    res.clearCookie(name);
  },
};
```

## Transformer Pattern

The transformer pattern keeps backend-specific response parsing isolated from the adapter logic:

```
Backend Response → Transformer → NormalizedUser / AuthResponse → Adapter → BFF
```

Benefits:
- Easy to test: transformers are pure functions
- Easy to update: when a backend changes its API, only the transformer needs updating
- Type safety: transformers bridge untyped backend responses to typed internal models

<Aside type="tip">
  Look at the existing transformer implementations for reference:
  - `apps/web/src/lib/adapters/laravel/transformer.ts` -- handles Laravel's `data` envelope
  - `apps/web/src/lib/adapters/symfony/transformer.ts` -- handles flat responses
  - `apps/web/src/lib/adapters/node/transformer.ts` -- handles camelCase field names
</Aside>

## Backend API Contract

Your custom backend must expose endpoints that the adapter calls. At minimum:

| Method | Endpoint | Request Body | Response |
| ------ | -------- | ------------ | -------- |
| `POST` | `/auth/login` | `{ email, password }` | `{ user, access_token, refresh_token, expires_in }` |
| `POST` | `/auth/register` | `{ name, email, password }` | Same as login |
| `GET` | `/auth/me` | -- | `{ user }` (with Bearer token) |
| `POST` | `/auth/refresh` | `{ refresh_token }` | Same as login |
| `POST` | `/auth/logout` | -- | `204 No Content` |

The exact field names and nesting can vary -- that is what the transformer handles.
