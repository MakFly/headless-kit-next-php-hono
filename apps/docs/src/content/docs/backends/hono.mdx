## Overview

The Hono backend (`apps/api-hono`) is a TypeScript backend running on Bun (port 3333). It uses **Vertical Slice Architecture** with a 5-file convention per feature.

| Feature | Details |
| ------- | ------- |
| Auth | JWT (HS256 via `jose`) |
| Architecture | VSA (`src/features/`, `src/shared/`) |
| Modules | Auth, Shop, Cart, Orders, Admin, SaaS, Support |
| Database | SQLite via Drizzle ORM |
| 2FA | No |
| OAuth | No |

## Setup

<Steps>

1. **Install dependencies**

   ```bash
   cd apps/api-hono
   bun install
   ```

2. **Configure environment**

   ```bash
   cp .env.example .env
   ```

3. **Run migrations and seed**

   ```bash
   bun run db:push
   bun run db:seed
   ```

4. **Start the server**

   ```bash
   bun run dev
   ```

</Steps>

### File Responsibilities

| File | Responsibility |
| ---- | ------------- |
| `*.schemas.ts` | Zod schemas for request validation |
| `*.repository.ts` | Database queries using Drizzle ORM |
| `*.service.ts` | Business logic, orchestration |
| `*.handlers.ts` | HTTP request/response handling |
| `*.routes.ts` | Hono route definitions with middleware |

## Authentication

JWT tokens signed with HS256:

| Method | Endpoint | Description |
| ------ | -------- | ----------- |
| `POST` | `/api/v1/auth/login` | Login |
| `POST` | `/api/v1/auth/register` | Register |
| `GET` | `/api/v1/auth/me` | Get current user |
| `POST` | `/api/v1/auth/refresh` | Refresh token |
| `POST` | `/api/v1/auth/logout` | Logout |

### Response Format

Hono uses camelCase:

```json
{
  "user": { "id": 1, "name": "John Doe", "email": "john@example.com" },
  "accessToken": "eyJ...",
  "refreshToken": "abc...",
  "expiresIn": 3600,
  "tokenType": "Bearer"
}
```

## Domain Modules

All 7 domain modules are implemented with the same 5-file convention: Shop, Cart, Orders, Admin, SaaS, and Support.
