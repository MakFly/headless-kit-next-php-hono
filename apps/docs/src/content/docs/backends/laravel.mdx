## Overview

The Laravel backend (`apps/api-laravel`) runs on port 8002 and is the most feature-complete backend. It uses **Vertical Slice Architecture** with all code organized under `app/Features/`.

| Feature | Details |
| ------- | ------- |
| Auth | BetterAuth (Bearer tokens) |
| Architecture | VSA (`app/Features/`, `app/Shared/`) |
| Modules | Auth, Shop, Cart, Orders, Admin, SaaS, Support, Users |
| Database | SQLite (dev) / PostgreSQL (prod) |
| 2FA | TOTP via BetterAuth |
| OAuth | Google, GitHub |
| RBAC | Full (roles, permissions, middleware) |

## Setup

<Steps>

1. **Install PHP dependencies**

   ```bash
   cd apps/api-laravel
   composer install
   ```

2. **Configure environment**

   ```bash
   cp .env.example .env
   php artisan key:generate
   ```

3. **Run migrations and seed**

   ```bash
   php artisan migrate --seed
   ```

4. **Start the server**

   ```bash
   php artisan serve --port=8002
   ```

</Steps>

## Authentication

BetterAuth handles all auth flows. Routes are registered under `/api/auth/`:

| Method | Endpoint | Description |
| ------ | -------- | ----------- |
| `POST` | `/api/auth/login` | Login with email/password |
| `POST` | `/api/auth/register` | Create new account |
| `GET` | `/api/auth/me` | Get current user with roles/permissions |
| `POST` | `/api/auth/refresh` | Refresh access token |
| `POST` | `/api/auth/logout` | Invalidate session |

### Response Format

Laravel wraps responses in a `data` envelope:

```json
{
  "data": {
    "user": { "id": 1, "name": "John Doe", "email": "john@example.com", "roles": [...], "permissions": [...] },
    "access_token": "eyJ...",
    "refresh_token": "abc...",
    "expires_in": 3600,
    "token_type": "Bearer"
  }
}
```

## Domain Modules

### Shop

| Method | Endpoint | Description |
| ------ | -------- | ----------- |
| `GET` | `/api/v1/shop/products` | List products (paginated) |
| `GET` | `/api/v1/shop/products/{id}` | Show product details |
| `GET` | `/api/v1/shop/categories` | List categories |
| `GET` | `/api/v1/shop/categories/{id}` | Show category with products |

### Cart

| Method | Endpoint | Description |
| ------ | -------- | ----------- |
| `GET` | `/api/v1/cart` | Get current cart |
| `POST` | `/api/v1/cart/items` | Add item to cart |
| `PATCH` | `/api/v1/cart/items/{id}` | Update item quantity |
| `DELETE` | `/api/v1/cart/items/{id}` | Remove item from cart |

### Orders

| Method | Endpoint | Description |
| ------ | -------- | ----------- |
| `POST` | `/api/v1/orders` | Create order from cart |
| `GET` | `/api/v1/orders` | List user's orders |
| `GET` | `/api/v1/orders/{id}` | Show order details |

### Admin

~30 actions covering RBAC management, product/order CRUD, customer management, review moderation, analytics, and inventory.

### SaaS

Organizations, plans, subscriptions, invoices, team management, usage tracking, and settings.

### Support

Conversations, messages, canned responses, agent queue, assignment, and satisfaction ratings.

## RBAC Middleware
