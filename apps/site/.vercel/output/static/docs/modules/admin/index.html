Admin & RBAC Module

Role-based access control with users, roles, permissions management and admin dashboard.

Admin & RBAC Module

The Admin module provides role-based access control (RBAC), user management, and a dashboard with analytics. All admin endpoints require authentication with the admin role.

RBAC Model

Roles

The system ships with three default roles:

RoleDescription
adminFull access to all admin endpoints
managerCan manage specific resources (configurable via permissions)
userStandard authenticated user (no admin access)

Permissions

Permissions follow a resource:action format:

  • users:manage — manage user accounts
  • products:create, products:update, products:delete — product CRUD
  • orders:manage — manage orders and statuses
  • reviews:moderate — approve/reject reviews

Permissions are assigned to roles, and roles are assigned to users. A user’s effective permissions are the union of all permissions from their assigned roles.

RBAC Endpoints

All endpoints under /api/v1/admin/ require auth + role:admin.

User Management

MethodEndpointDescription
GET/api/v1/admin/usersList all users (paginated)
GET/api/v1/admin/users/{id}Get user detail with roles

Role Management

MethodEndpointDescription
GET/api/v1/admin/rolesList all roles
POST/api/v1/admin/rolesCreate a new role
POST/api/v1/admin/users/{id}/rolesAssign role to user
DELETE/api/v1/admin/users/{id}/roles/{role}Remove role from user

Permission Management

MethodEndpointDescription
GET/api/v1/admin/permissionsList all available permissions
POST/api/v1/admin/roles/{role}/permissionsSync permissions for a role

Dashboard Endpoints

MethodEndpointDescription
GET/api/v1/admin/dashboardSummary stats (revenue, orders, users, etc.)
GET/api/v1/admin/analytics/revenueRevenue over time
GET/api/v1/admin/analytics/top-productsBest selling products
GET/api/v1/admin/segmentsCustomer segments

Implementation per Backend

Laravel (apps/api-laravel/)

RBAC is enforced via middleware aliases defined in bootstrap/app.php:

MiddlewareClassPurpose
role:adminApp\Shared\Middleware\CheckRoleRequires the admin role
permission:XApp\Shared\Middleware\CheckPermissionRequires a specific permission

All admin routes are grouped under Route::middleware(['auth:betterauth', 'role:admin'])->prefix('admin') in app/Features/Admin/routes.php.

Key files:

  • Routes: app/Features/Admin/routes.php
  • Actions: app/Features/Admin/Actions/ (ListUsers, ShowUser, AssignRole, RemoveRole, CreateRole, ListRoles, ListPermissions, SyncPermissions, Dashboard, RevenueAnalytics, TopProducts, ListSegments)
  • Models: app/Shared/Models/User.php, app/Shared/Models/Role.php, app/Shared/Models/Permission.php

Symfony (apps/api-sf/)

RBAC uses Symfony’s Voter system and the firewall security layer:

  • Admin routes require ROLE_ADMIN via #[IsGranted('ROLE_ADMIN')] or firewall config
  • Organization-level authorization uses custom Voters

Key files:

  • RBAC controllers: src/Feature/Admin/Controller/Rbac/ (6 controllers: AssignRoleController, CreateRoleController, GetUserRolesController, ListPermissionsController, ListRolesController, RemoveRoleController)
  • Shop admin controllers: src/Feature/Admin/Controller/Shop/ (8 controllers)
  • Segment service: src/Feature/Admin/Service/SegmentQueryService.php
  • Security: src/Shared/Security/ (Voters, traits)

Hono (apps/api-hono/)

RBAC uses the adminMiddleware which checks the user’s role from the JWT payload:

admin.use('*', authMiddleware);
admin.use('*', adminMiddleware);

Key files:

  • Feature: src/features/admin/ (admin.routes.ts, admin.handlers.ts, admin.service.ts, admin.repository.ts, admin.schemas.ts)
  • Middleware: src/shared/middleware/admin.ts

Data Tables

The admin pages use TanStack Table for data grids with server-side pagination, sorting, and filtering. Tables are configured per resource with column definitions, filter controls, and action menus.
