Shop Module

E-commerce module with products, categories, cart, orders, and admin management across all backends.

Shop Module

The Shop module provides a complete e-commerce flow: product catalog, categories, cart management, checkout, and order tracking. Admin endpoints cover product CRUD, order management, inventory, customer management, reviews, and analytics.

Public Endpoints

Products

MethodEndpointDescription
GET/api/v1/productsList products (paginated, filterable by category, price range, search)
GET/api/v1/products/{slug}Get product detail by slug

Query parameters: ?page=1&per_page=20&category=shoes&min_price=10&max_price=100&search=keyword&sort=-created_at

Categories

MethodEndpointDescription
GET/api/v1/categoriesList all categories
GET/api/v1/categories/{slug}Get category with its products

Authenticated Endpoints

Cart

All cart endpoints require authentication.

MethodEndpointDescription
GET/api/v1/cartGet current user’s cart with items
POST/api/v1/cart/itemsAdd item to cart (productId, quantity)
PATCH/api/v1/cart/items/{id}Update cart item quantity
DELETE/api/v1/cart/items/{id}Remove item from cart

Orders

MethodEndpointDescription
POST/api/v1/ordersCreate order from current cart
GET/api/v1/ordersList user’s orders (paginated)
GET/api/v1/orders/{id}Get order detail with items

Admin Endpoints

All admin endpoints require auth + role:admin (Laravel/Hono) or IS_AUTHENTICATED_FULLY + ROLE_ADMIN (Symfony).

Product Management

MethodEndpointDescription
GET/api/v1/admin/productsList all products (with admin details)
POST/api/v1/admin/productsCreate product
PUT/api/v1/admin/products/{id}Update product
DELETE/api/v1/admin/products/{id}Delete product

Order Management

MethodEndpointDescription
GET/api/v1/admin/ordersList all orders
PATCH/api/v1/admin/orders/{id}/statusUpdate order status

Inventory

MethodEndpointDescription
GET/api/v1/admin/inventoryList product inventory
PATCH/api/v1/admin/inventory/{productId}Update stock quantity

Customers

MethodEndpointDescription
GET/api/v1/admin/customersList customers
POST/api/v1/admin/customersCreate customer
GET/api/v1/admin/customers/{id}Get customer detail
PUT/api/v1/admin/customers/{id}Update customer
DELETE/api/v1/admin/customers/{id}Delete customer

Reviews

MethodEndpointDescription
GET/api/v1/admin/reviewsList all reviews
GET/api/v1/admin/reviews/{id}Get review detail
PUT/api/v1/admin/reviews/{id}Update review (approve/reject)
POST/api/v1/admin/reviews/bulk-approveBulk approve reviews
POST/api/v1/admin/reviews/bulk-rejectBulk reject reviews

Dashboard and Analytics

MethodEndpointDescription
GET/api/v1/admin/dashboardDashboard summary (revenue, orders, customers)
GET/api/v1/admin/analytics/revenueRevenue analytics over time
GET/api/v1/admin/analytics/top-productsTop selling products
GET/api/v1/admin/segmentsCustomer segments

Implementation per Backend

Laravel (apps/api-laravel/)

FeaturePath
Shop routesapp/Features/Shop/routes.php
Shop actionsapp/Features/Shop/Actions/ (ListProducts, ShowProduct, ListCategories, ShowCategory)
Cart routesapp/Features/Cart/routes.php
Cart actionsapp/Features/Cart/Actions/ (ShowCart, AddItem, UpdateItem, RemoveItem)
Cart formatterapp/Features/Cart/Formatters/CartFormatter.php
Order routesapp/Features/Orders/routes.php
Order actionsapp/Features/Orders/Actions/ (CreateOrder, ListOrders, ShowOrder)
Admin routesapp/Features/Admin/routes.php
Admin actionsapp/Features/Admin/Actions/ (~25 action classes)
Modelsapp/Shared/Models/ (Product, Category, Cart, CartItem, Order, OrderItem)

Symfony (apps/api-sf/)

FeaturePath
ShopAPI Platform resources on Product and Category entities (no controllers)
Cart controllerssrc/Feature/Cart/Controller/ (ShowCartController, AddItemController, etc.)
Cart repositorysrc/Feature/Cart/Repository/CartRepository.php
Order controllersrc/Feature/Orders/Controller/ (CreateOrder; List/Show via API Platform)
Admin controllerssrc/Feature/Admin/Controller/Shop/ (8 controllers)
Entitiessrc/Shared/Entity/ (Product, Category, Order, OrderItem, etc.)

Hono (apps/api-hono/)

FeaturePath
Shopsrc/features/shop/ (5 files: routes, handlers, service, repository, schemas)
Cartsrc/features/cart/ (5 files)
Orderssrc/features/orders/ (5 files)
Adminsrc/features/admin/ (5 files)
DB schemasrc/shared/db/schema.ts (products, categories, carts, cart_items, orders, order_items)

Server Actions

Located in apps/web/src/lib/actions/shop/:

FileActions
products.tsFetch products, product detail, categories
cart.tsGet cart, add/update/remove items
orders.tsCreate order, list orders, get order detail
admin.tsAdmin product CRUD, order management, inventory, customers, reviews, analytics

All Server Actions use the BFF client (apps/web/src/lib/actions/_shared/bff-client.ts) — never direct fetch() calls to backends.
