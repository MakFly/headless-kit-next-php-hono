## Overview

The Hono backend (`apps/api-hono`) is a TypeScript backend running on Bun (port 8003). It uses **Vertical Slice Architecture** with a 5-file convention per feature.

| Feature | Details |
| ------- | ------- |
| Auth | JWT (HS256 via `jose`) |
| Architecture | VSA (`src/features/`, `src/shared/`) |
| Modules | Auth, Shop, Cart, Orders, Admin, SaaS, Support |
| Database | SQLite via Drizzle ORM |
| 2FA | No |
| OAuth | No |

## Setup

<Steps>

1. **Install dependencies**

   ```bash
   cd apps/api-hono
   bun install
   ```

2. **Configure environment**

   ```bash
   cp .env.example .env
   ```

3. **Run migrations and seed**

   ```bash
   bun run db:migrate
   bun run db:seed
   ```

4. **Start the server**

   ```bash
   bun run dev
   ```

</Steps>

## Project Structure — 5-File Convention

Each feature follows a strict 5-file convention:

```
src/features/
├── auth/
│   ├── auth.schemas.ts       # Zod validation schemas
│   ├── auth.repository.ts    # Database queries (Drizzle)
│   ├── auth.service.ts       # Business logic
│   ├── auth.handlers.ts      # Request handlers
│   └── auth.routes.ts        # Hono route definitions
├── shop/
│   ├── shop.schemas.ts
│   ├── shop.repository.ts
│   ├── shop.service.ts
│   ├── shop.handlers.ts
│   └── shop.routes.ts
├── cart/                     # Same 5-file pattern
├── orders/                   # Same 5-file pattern
├── admin/                    # Same 5-file pattern
├── saas/                     # Same 5-file pattern
└── support/                  # Same 5-file pattern

src/shared/
├── db/                       # Drizzle schema, migrations
├── middleware/                # Auth guard, CORS, security
└── lib/                      # JWT utilities, hash helpers
```

### File Responsibilities

| File | Responsibility |
| ---- | ------------- |
| `*.schemas.ts` | Zod schemas for request validation |
| `*.repository.ts` | Database queries using Drizzle ORM |
| `*.service.ts` | Business logic, orchestration |
| `*.handlers.ts` | HTTP request/response handling |
| `*.routes.ts` | Hono route definitions with middleware |

## Authentication

JWT tokens signed with HS256:

| Method | Endpoint | Description |
| ------ | -------- | ----------- |
| `POST` | `/api/v1/auth/login` | Login |
| `POST` | `/api/v1/auth/register` | Register |
| `GET` | `/api/v1/auth/me` | Get current user |
| `POST` | `/api/v1/auth/refresh` | Refresh token |
| `POST` | `/api/v1/auth/logout` | Logout |

### Response Format

Hono uses camelCase:

```json
{
  "user": { "id": 1, "name": "John Doe", "email": "john@example.com" },
  "accessToken": "eyJ...",
  "refreshToken": "abc...",
  "expiresIn": 3600,
  "tokenType": "Bearer"
}
```
