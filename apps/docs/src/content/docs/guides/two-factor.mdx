## Overview

Two-factor authentication (2FA) is available on **Laravel** and **Symfony** backends. It uses TOTP (Time-based One-Time Passwords) compatible with apps like Google Authenticator, Authy, or 1Password.

<Aside>
  The Hono backend does not support 2FA. If you need 2FA, use Laravel or Symfony.
</Aside>

## Setup Flow

<Steps>

1. **Check 2FA status**

   ```typescript
   const status = await bff.get('/api/v1/auth/2fa/status');
   // { enabled: false }
   ```

2. **Request TOTP setup**

   ```typescript
   const setup = await bff.post('/api/v1/auth/2fa/setup');
   // {
   //   secret: "JBSWY3DPEHPK3PXP",
   //   qr_uri: "otpauth://totp/HeadlessKit:user@example.com?secret=JBSWY3DPEHPK3PXP&issuer=HeadlessKit",
   //   recovery_codes: ["abc123", "def456", ...]
   // }
   ```

   Display the `qr_uri` as a QR code for the user to scan with their authenticator app. The `recovery_codes` should be shown once and stored securely by the user.

3. **Verify and enable**

   After the user scans the QR code, they enter the 6-digit code from their authenticator:

   ```typescript
   await bff.post('/api/v1/auth/2fa/verify', {
     code: '123456',
   });
   // 2FA is now enabled for this user
   ```

4. **Store recovery codes**

   Remind the user to save their recovery codes. These are one-time-use codes that allow login if the authenticator app is unavailable.

</Steps>

## Login with 2FA

When 2FA is enabled, the login flow changes:

<Steps>

1. **Submit credentials**

   ```typescript
   const result = await bff.post('/api/v1/auth/login', {
     email: 'user@example.com',
     password: 'password',
   });
   ```

2. **Check for 2FA challenge**

   If the user has 2FA enabled, the response includes a `requires_2fa` flag:

   ```json
   {
     "requires_2fa": true,
     "two_factor_token": "temporary-token-string"
   }
   ```

   No access token is returned at this stage. The `two_factor_token` is a short-lived token for the 2FA verification step.

3. **Submit TOTP code**

   ```typescript
   const authResult = await bff.post('/api/v1/auth/2fa/challenge', {
     two_factor_token: result.two_factor_token,
     code: '123456', // from authenticator app
   });
   // Returns full auth response with access_token
   ```

   Alternatively, the user can use a recovery code:

   ```typescript
   const authResult = await bff.post('/api/v1/auth/2fa/challenge', {
     two_factor_token: result.two_factor_token,
     recovery_code: 'abc123',
   });
   ```

</Steps>

## Disabling 2FA

```typescript
await bff.post('/api/v1/auth/2fa/disable', {
  password: 'current-password', // re-authentication required
});
```

## Backend Configuration

<Tabs>
  <TabItem label="Laravel">
    2FA is built into BetterAuth for Laravel. No additional configuration is needed beyond having BetterAuth installed.

    The TOTP secret is stored encrypted in the `users` table (or a related `two_factor` table depending on the BetterAuth version).

    Relevant endpoints:

    | Method | Endpoint | Description |
    | ------ | -------- | ----------- |
    | `GET` | `/api/auth/2fa/status` | Check if 2FA is enabled |
    | `POST` | `/api/auth/2fa/setup` | Generate secret + QR URI |
    | `POST` | `/api/auth/2fa/verify` | Verify code and enable |
    | `POST` | `/api/auth/2fa/disable` | Disable 2FA |
    | `POST` | `/api/auth/2fa/challenge` | Verify during login |
  </TabItem>
  <TabItem label="Symfony">
    The `betterauth/symfony-bundle` handles 2FA with the same endpoint pattern as Laravel.

    The TOTP secret is stored in the user entity and encrypted using `BETTER_AUTH_SECRET`.

    Make sure your `.env.local` has a strong `BETTER_AUTH_SECRET` value, as it is used both for Paseto token signing and TOTP secret encryption.
  </TabItem>
</Tabs>

## Frontend Implementation Tips

- Show the QR code using a library like `qrcode.react` or generate an `<img>` from the `qr_uri`
- Store the 2FA state in your login flow (e.g., `'credentials' | '2fa-challenge'`)
- Always show recovery codes on setup and allow the user to download/copy them
- Consider adding a "Remember this device" option in your frontend if needed
