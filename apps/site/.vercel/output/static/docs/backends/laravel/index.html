Laravel Backend

Laravel 12 API backend with BetterAuth, Vertical Slice Architecture, and API Platform OpenAPI docs.

Laravel Backend

The Laravel backend (apps/api-laravel/) runs on port 8002 and provides a full-featured REST API using Laravel 12, BetterAuth for authentication, and Eloquent ORM with SQLite.

Setup

cd apps/api-laravel
composer install
cp .env.example .env
php artisan key:generate
php artisan migrate --seed
php artisan serve --port=8002

The API is now available at http://localhost:8002/api/v1/.

Project Structure

The codebase follows Vertical Slice Architecture (VSA). Each feature is self-contained under app/Features/:

app/
├── Features/
│   ├── Auth/          # Authentication (BetterAuth + custom actions)
│   ├── Shop/          # Products & categories (public)
│   ├── Cart/          # Cart management (authenticated)
│   ├── Orders/        # Order management (authenticated)
│   ├── Users/         # User endpoints
│   ├── Admin/         # Admin CRUD + analytics (role:admin)
│   ├── Saas/          # Multi-tenant SaaS (org.rbac)
│   └── Support/       # Helpdesk & conversations
├── Shared/
│   ├── Models/        # ALL Eloquent models (User, Product, Order, Cart...)
│   ├── Middleware/     # CheckRole, CheckPermission, OrgRbacMiddleware
│   ├── Helpers/       # ApiResponse standardized responses
│   └── Traits/        # ApiResponder trait

Each feature directory contains:

DirectoryPurpose
Actions/Invokable Action classes (1 per endpoint)
Requests/Form Request validation (optional)
Services/Business logic (optional)
Formatters/Static format methods (optional)
routes.phpFeature route definitions

All feature routes are included inside the Route::prefix('v1') group in routes/api.php.

Authentication

Laravel uses BetterAuth (betterauth/laravel) with Bearer tokens. BetterAuth handles the core auth routes natively; custom endpoints (password reset, 2FA, OAuth) are added as invokable Action classes.

Auth Endpoints

POST/api/v1/auth/registerPublicRegister a new user
POST/api/v1/auth/loginPublicLogin (returns access + refresh tokens)
POST/api/v1/auth/refreshPublicRefresh access token
GET/api/v1/auth/meBearerGet current user
PATCH/api/v1/auth/meBearerUpdate profile
DELETE/api/v1/auth/meBearerDelete account
POST/api/v1/auth/logoutBearerLogout
POST/api/v1/auth/revoke-allBearerRevoke all sessions
PUT/api/v1/auth/passwordBearerUpdate password

Response Format

All responses use the ApiResponse helper with a consistent envelope:

{
  "success": true,
  "data": { "..." },
  "status": 200,
  "request_id": "abc123"
}

Error responses:

{
  "success": false,
  "error": { "code": "VALIDATION_ERROR", "message": "..." },
  "status": 422
}

Testing

php artisan test                            # Run all tests
php artisan test tests/Feature/ShopTest.php # Specific test file
php artisan test --filter=test_name         # Single test

Test files in tests/Feature/: BetterAuthTest.php, ShopTest.php, CartTest.php, OrderTest.php, AdminShopTest.php, SaasTest.php, SupportTest.php.

QA

make qa-laravel    # Runs Pint (formatter) + Larastan (static analysis) + PHPUnit

API Platform

api-platform/laravel is used exclusively as an OpenAPI documentation source. BetterAuth routes remain as classic Laravel routes and are never migrated to API Platform resources.
