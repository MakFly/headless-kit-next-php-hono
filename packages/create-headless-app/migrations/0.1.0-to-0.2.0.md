# Migration Guide: v0.1.0 → v0.2.0

## Who needs this?

Projects scaffolded with `create-headless-app` v0.1.0. Check your `README.md` for the version or look for the absence of `v{{HEADLESS_KIT_VERSION}}` in your README.

## Breaking Changes

### 1. Port numbers changed

v0.1.0 used incorrect ports in `.env` files. Update these:

**Frontend `.env.local`:**
```diff
- NEXT_PUBLIC_APP_URL=http://localhost:3001
+ NEXT_PUBLIC_APP_URL=http://localhost:3300

- VITE_APP_URL=http://localhost:3003
+ VITE_APP_URL=http://localhost:3301
```

**Frontend `package.json` scripts:**
```diff
- "dev": "next dev --port 3001"
+ "dev": "next dev --port 3300"

- "dev": "vite dev --port 3003"
+ "dev": "vite dev --port 3301"
```

**Backend `.env`:**
```diff
# Laravel
- APP_URL=http://localhost:8000
+ APP_URL=http://localhost:8002

# Hono
- PORT=8003
+ PORT=3333
```

### 2. Laravel auth routes moved to /api/v1

If you're using the **Laravel** backend, auth routes moved from `/api/auth/*` to `/api/v1/auth/*`.

**Update `apps/web/src/lib/adapters/laravel/adapter.ts`** (if you haven't customized it):
```diff
const ENDPOINTS = {
-  LOGIN: "/api/auth/login",
+  LOGIN: "/api/v1/auth/login",
   // ... same for all other endpoints
};
```

**Update `apps/web/src/lib/adapters/proxy-config.ts`** — the `getLaravelConfig` path transform is no longer needed (1:1 passthrough like Symfony).

### 3. Unused backend adapters

v0.2.0 now removes adapters for backends you didn't select. If you want this on an existing project, manually delete:
- `src/lib/adapters/laravel/` (if not using Laravel)
- `src/lib/adapters/symfony/` (if not using Symfony)
- `src/lib/adapters/node/` (if not using Hono)

Then update `src/lib/adapters/index.ts` to only import your selected adapter.

## New Features (opt-in)

### Security headers

Add to your backend:

**Laravel** — `app/Shared/Middleware/ApiSecurityHeadersMiddleware.php`:
```php
$response->headers->set('X-Frame-Options', 'SAMEORIGIN');
$response->headers->set('X-Content-Type-Options', 'nosniff');
$response->headers->set('Strict-Transport-Security', 'max-age=31536000; includeSubDomains');
$response->headers->set('X-XSS-Protection', '0');
$response->headers->set('Referrer-Policy', 'strict-origin-when-cross-origin');
```

Register globally in `bootstrap/app.php`.

**Symfony** — create `src/Shared/EventSubscriber/SecurityHeadersSubscriber.php` (KernelEvents::RESPONSE, scope `/api/`). Same headers as above.

**Hono** — already has `secureHeaders()` globally. No action needed.

### JWT production guard (Hono only)

In `src/shared/lib/jwt.ts`, add a startup check:
```typescript
const rawSecret = process.env.JWT_SECRET || 'dev-secret';
if (process.env.NODE_ENV === 'production' && (rawSecret === 'dev-secret' || rawSecret.length < 32)) {
  throw new Error('JWT_SECRET must be set to a secure value (min 32 chars) in production');
}
```

### ai-assistant module

If you want to add AI chat to an existing project, copy these from a freshly scaffolded project (with ai-assistant module):
- `src/app/api/ai/chat/route.ts` (Next.js) or `src/routes/api/ai/chat.ts` (TanStack)
- `src/components/assistant-ui/` (6 files)
- `src/lib/ai/providers.ts` + `provider-options.ts`
- `src/stores/ai-preferences-store.ts`

Add dependencies:
```bash
bun add ai @ai-sdk/anthropic @ai-sdk/openai @ai-sdk/google @ai-sdk/mistral @ai-sdk/react @assistant-ui/react @assistant-ui/react-ai-sdk @assistant-ui/react-markdown remark-gfm
```
