## Overview

Headless Kit can be deployed using Docker Compose with separate containers for each service. Each backend has its own Dockerfile. Both the Next.js BFF and TanStack Start frontend can be containerized.

## Docker Compose Structure

```yaml
# docker-compose.yml
services:
  web:
    build:
      context: .
      dockerfile: apps/web/Dockerfile
    ports:
      - "3300:3300"
    environment:
      - NEXT_PUBLIC_APP_URL=http://localhost:3300
      - AUTH_BACKEND=laravel
      - LARAVEL_API_URL=http://api-laravel:8002
      - SYMFONY_API_URL=http://api-sf:8001
      - HONO_API_URL=http://api-hono:3333
      - BFF_SECRET=${BFF_SECRET}
      - BFF_ID=nextjs-bff
    depends_on:
      - api-laravel

  web-tanstack:
    build:
      context: .
      dockerfile: apps/web-tanstack/Dockerfile
    ports:
      - "3301:3301"
    environment:
      - VITE_API_URL=http://api-hono:3333
      - VITE_APP_URL=http://localhost:3301
    depends_on:
      - api-hono

  api-laravel:
    build:
      context: .
      dockerfile: apps/api-laravel/Dockerfile
    ports:
      - "8002:8002"
    environment:
      - APP_URL=http://localhost:8002
      - APP_KEY=${LARAVEL_APP_KEY}
      - DB_CONNECTION=sqlite
      - DB_DATABASE=/app/database/database.sqlite
      - BFF_SECRET=${BFF_SECRET}
      - BFF_ID=nextjs-bff
      - FRONTEND_URL=http://localhost:3300
    volumes:
      - laravel-data:/app/database

  api-sf:
    build:
      context: .
      dockerfile: apps/api-sf/Dockerfile
    ports:
      - "8001:8001"
    environment:
      - DATABASE_URL=sqlite:///app/var/data.db
      - BETTER_AUTH_SECRET=${BETTER_AUTH_SECRET}
      - FRONTEND_URL=http://localhost:3300
    volumes:
      - symfony-data:/app/var

  api-hono:
    build:
      context: .
      dockerfile: apps/api-hono/Dockerfile
    ports:
      - "3333:3333"
    environment:
      - PORT=3333
      - JWT_SECRET=${JWT_SECRET}
      - FRONTEND_URL=http://localhost:3301
      - DATABASE_URL=file:/app/data/dev.db
    volumes:
      - hono-data:/app/data

volumes:
  laravel-data:
  symfony-data:
  hono-data:
```

## Dockerfiles

### Next.js BFF

```dockerfile
# apps/web/Dockerfile
FROM oven/bun:1 AS base
WORKDIR /app

# Install dependencies
FROM base AS deps
COPY package.json bun.lock ./
COPY apps/web/package.json apps/web/
RUN bun install --frozen-lockfile

# Build
FROM base AS builder
COPY --from=deps /app/node_modules ./node_modules
COPY . .
RUN bun run --filter @headless/web build

# Production
FROM base AS runner
ENV NODE_ENV=production
COPY --from=builder /app/apps/web/.next/standalone ./
COPY --from=builder /app/apps/web/.next/static ./apps/web/.next/static
COPY --from=builder /app/apps/web/public ./apps/web/public

EXPOSE 3300
CMD ["bun", "apps/web/server.js"]
```

### Laravel

```dockerfile
# apps/api-laravel/Dockerfile
FROM php:8.3-fpm AS base

RUN apt-get update && apt-get install -y \
    libsqlite3-dev \
    && docker-php-ext-install pdo_sqlite

COPY --from=composer:2 /usr/bin/composer /usr/bin/composer

WORKDIR /app
COPY apps/api-laravel/ .
RUN composer install --no-dev --optimize-autoloader

RUN php artisan config:cache && \
    php artisan route:cache

EXPOSE 8002
CMD ["php", "artisan", "serve", "--host=0.0.0.0", "--port=8002"]
```

### TanStack Start

```dockerfile
# apps/web-tanstack/Dockerfile
FROM oven/bun:1 AS base
WORKDIR /app

# Install dependencies
FROM base AS deps
COPY package.json bun.lock ./
COPY apps/web-tanstack/package.json apps/web-tanstack/
RUN bun install --frozen-lockfile

# Build
FROM base AS builder
COPY --from=deps /app/node_modules ./node_modules
COPY . .
RUN bun run --filter @headless/web-tanstack build

# Production
FROM base AS runner
ENV NODE_ENV=production
COPY --from=builder /app/apps/web-tanstack/.output ./

EXPOSE 3301
CMD ["bun", "run", "server/index.mjs"]
```

### Hono

```dockerfile
# apps/api-hono/Dockerfile
FROM oven/bun:1 AS base
WORKDIR /app

COPY apps/api-hono/package.json ./
RUN bun install --production

COPY apps/api-hono/ .

EXPOSE 3333
CMD ["bun", "run", "src/index.ts"]
```

## Environment Variables

Create a `.env` file at the project root for Docker Compose:

```bash
# .env (root, for Docker Compose)
BFF_SECRET=your-production-hmac-secret-min-32-chars
LARAVEL_APP_KEY=base64:your-laravel-app-key
BETTER_AUTH_SECRET=your-symfony-auth-secret
JWT_SECRET=your-hono-jwt-secret
```

<Aside type="caution">
  Never commit this file. Add `.env` to `.gitignore` and use a secrets manager (e.g., Docker Secrets, Vault, AWS SSM) in production.
</Aside>

## Running with Docker Compose

```bash
# Build and start all services
docker compose up --build

# Start only specific services
docker compose up web api-laravel

# Run in the background
docker compose up -d

# View logs
docker compose logs -f web

# Stop all services
docker compose down
```

## Database Persistence

Each backend uses a named volume for its SQLite database:

| Service | Volume | Path in Container |
| ------- | ------ | ----------------- |
| Laravel | `laravel-data` | `/app/database/` |
| Symfony | `symfony-data` | `/app/var/` |
| Hono | `hono-data` | `/app/data/` |

To reset a database, remove the volume:

```bash
docker compose down -v  # removes all volumes
docker volume rm headless-kit_laravel-data  # remove specific volume
```

## Running Migrations in Docker

```bash
# Laravel
docker compose exec api-laravel php artisan migrate --seed

# Symfony
docker compose exec api-sf php bin/console doctrine:migrations:migrate

# Hono
docker compose exec api-hono bun run db:migrate
docker compose exec api-hono bun run db:seed
```
